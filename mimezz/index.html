MiMezz

Enabling Mezzanine design for everyone

The Challenge

Make the complex simple

MiMezz is a tool to help Sales Associates design preliminary mezzanine layouts for clients to obtain an initial costing. Since Sales Associates are not Architects this means MiMezz needs to be as simple as possible so that it’s easy for Sales Associates to achieve their goals.

A mezzanine is an elevated structure that is installed inside a warehouse which allows warehouse operators to increase their storage in a cost efficient manner.

The Solution

Select to draw

Although MiMezz can help you design a Mezzanine from the ground up, my focus on the project was Zone Framing. To effectively design a mezzanine in a cost efficient manner the system needs to be told by the user in what way to divide the mezzanine up, which we call Zones, and what properties each Zone should have.

For MiMezz’s intended audience (Sales Associates) there is a balancing act between providing them with enough flexibility to create any mezzanine their client desires and subtly guiding them in ways so they avoid creating mezzanines that might be uneconomical or have invalid properties.

To handle this balance I came up with a drawing system with similar properties to Microsoft Excel’s cell selection tool. When you create a mezzanine MiMezz will automatically create Build Lines from the frame of the mezzanine. These Build Lines just so happen to make a grid with cells. When you want to add a new Zone you can simply click and drag to select the number of cells you need for your Zone and then adjust its’ properties. When comparing this drawing method to something more freeform this feels much more intuitive, like the system knows what you want.

The Process

Getting the right feel

The Zone Framing feature went through many iterations to come up with a solution that felt intuitive to use. It was important to get this right

Iteration 1

Focused on the idea of using the auto-generated zones and connecting them together. While this would have been really simple it lacked the flexibility that users needed.

Iteration 2

If users needed flexibility then how about letting them draw any arbitrary zone? While this enabled flexibility it could have easily resulted in sub-optimal zone layouts.

Iteration 3

This idea builds on iteration 2 by allowing the draw tool to snap to build lines that the system has determined would be valid. It also introduces the idea of grouped zones where the properties of one zone flow in to another contiguous zone.

Iteration 4

We start getting closer here. Zones now show their number of bays in a way that links them across with other zones. The dotted lines are intended to show how everything flows together but it became quite complex.

Iteration 5

A concern was having bays line up and what happens when they don’t. Here I introduced a complex locking and directionality system that allowed the user to determine what zones are considered part of the same group and in what ways the number of bays should flow.

There was also a concern we were asking users to draw their mezzanine twice so I had to go back to the drawing board. I came up with the idea of using the cell selection method from Excel since we already had build lines that are generated. This method not only felt right it made it incredibly quick to mark out your zones.

Iteration 6

We realised that the locking and orientation system isn’t needed if instead we have the concept of a “Key Zone” and all contiguous zones inherit the properties of the Key Zone. If a contiguous bay is modified it becomes it’s on Key Zone and any subsequent bays will inherit the properties of the new Key Zone.
